# Internationalization (i18n)

## Overview

The app uses [next-intl](https://next-intl-docs.vercel.app/) for internationalization. Currently supports English (en) and Portuguese (pt). Locale is automatically detected from the browser's `Accept-Language` header and can be manually switched via the UI language picker.

## Architecture

**Framework:** next-intl v4.11.0 with Next.js 16 App Router

**Key files:**
- `messages/{locale}.json` - Translation strings for each locale
- `src/i18n/config.ts` - Locale list, display names, flags
- `src/i18n/routing.ts` - Routing configuration
- `src/i18n/request.ts` - Server-side locale resolution
- `src/middleware.ts` - Automatic locale detection
- `src/app/[locale]/` - Localized route pages
- `src/components/LanguageSwitcher.tsx` - Manual language picker UI

**URL structure:**
- `/` → Default locale (English)
- `/pt` → Portuguese
- `/en/match/123` → English match page
- `/pt/match/123` → Portuguese match page

The `localePrefix: 'as-needed'` strategy means the default locale (English) works at `/` or `/en`, while other locales require the prefix (`/pt`).

## Adding a new language

### 1. Create translation file

Copy `messages/en.json` to `messages/{locale}.json` (e.g., `es.json` for Spanish). Translate all values; keep keys unchanged.

### 2. Update locale configuration

Edit `src/i18n/config.ts`:

```typescript
export const locales = ['en', 'pt', 'es'] as const;

export const localeNames: Record<Locale, string> = {
  en: 'English',
  pt: 'Português',
  es: 'Español',
};

export const localeFlags: Record<Locale, string> = {
  en: '🇺🇸',
  pt: '🇧🇷',
  es: '🇪🇸',
};
```

### 3. Update routing

Edit `src/i18n/routing.ts`:

```typescript
export const routing = defineRouting({
  locales: ['en', 'pt', 'es'],
  defaultLocale: 'en',
  localePrefix: 'as-needed',
});
```

### 4. Update language switcher

Edit `src/components/LanguageSwitcher.tsx` - add your locale to the `localeNames` and `localeFlags` objects (same values as step 2).

### 5. Test

```bash
npm run dev
# Visit http://localhost:3000/{locale} to test
```

## Translation conventions

**What is translated:**
- All UI strings (buttons, labels, headings, error messages)
- Page metadata (titles, descriptions, OpenGraph tags)

**What is NOT translated:**
- AI-generated card clues (controlled by API locale parameter)
- Share result text format (intentionally English for universal readability)
- Code, file names, configuration values

Use the `useTranslations` hook in components:

```tsx
import { useTranslations } from 'next-intl';

export default function MyComponent() {
  const t = useTranslations('namespace');
  return <button>{t('buttonKey')}</button>;
}
```

Translation keys use dot notation (`"namespace.section.key"`). Organize by feature or component.
